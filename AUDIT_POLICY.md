# OpenClaw 文章发布审核制度

## 概述

为确保发布文章的质量、准确性和合规性，建立本审核制度。所有文章必须经过审核机器人检查，**审核通过后才能发布**。

---

## 审核流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  撰写文章   │ → │  审核检查   │ → │  审核通过   │ → │  正式发布   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                          │
                          ↓ 未通过
                   ┌─────────────┐
                   │  修复问题   │
                   └─────────────┘
```

---

## 审核标准

### 1. 数据准确性（强制）

**关键财务数据必须核实：**

| 公司 | 关键指标 | 核实来源 |
|------|---------|---------|
| NVIDIA | 营收、EPS、数据中心收入 | 官方财报、CNBC、Investopedia |
| Tesla | 交付量、营收 | 官方公告 |
| 各大公司 | 财报数据 | SEC filing、官方公告 |

**错误示例：**
- ❌ NVIDIA总营收$393亿（实际$681亿）
- ❌ 未标注数据来源

**正确示例：**
- ✅ NVIDIA总营收$681.3亿（同比增长73%）
- ✅ 数据来源：NVIDIA官方财报

### 2. 格式规范（强制）

**必须包含：**
- [ ] 一级标题（文章标题）
- [ ] 发布时间
- [ ] 来源声明
- [ ] 免责声明
- [ ] 字数≥1500字

**禁止事项：**
- [ ] 代码块（```）
- [ ] JSON花括号（可能导致微信解析错误）
- [ ] 敏感政治词汇

### 3. 内容结构（建议）

**推荐结构：**
```
# 文章标题
**发布时间：**
**来源声明：**

---

## 一、核心动态
## 二、深度分析
## 三、数据对比
## 四、投资建议
## 五、风险提示

---

**免责声明**
```

### 4. 安全检查（强制）

- [ ] 无敏感政治内容
- [ ] 无违法信息
- [ ] 无侵权内容
- [ ] 数据来源合法

## 审核命令

### 单篇文章审核

```bash
# 审核单篇文章
python3 /root/.openclaw/workspace/audit_bot.py 文章路径.md

# 示例
python3 /root/.openclaw/workspace/audit_bot.py \
  /root/.openclaw/workspace/news/articles/20260226-NVIDIA财报.md
```

### 批量审核

```bash
# 审核所有未发布文章
for f in /root/.openclaw/workspace/news/articles/*.md; do
  echo "审核: $f"
  python3 /root/.openclaw/workspace/audit_bot.py "$f"
  echo "---"
done
```

## 发布前检查清单

### 人工复核（最后把关）

**数据核实：**
- [ ] 所有财务数据已核实
- [ ] 关键数字已二次确认
- [ ] 数据来源已标注

**格式检查：**
- [ ] 标题层级正确
- [ ] 表格格式正确
- [ ] 无代码块/花括号
- [ ] 图片已设置

**内容检查：**
- [ ] 语句通顺
- [ ] 逻辑清晰
- [ ] 无错别字
- [ ] 无敏感内容

## 审核结果处理

### 审核通过 ✅

```
✅ 审核结果: 通过
文章可以发布到公众号
```

### 有条件通过 🟡

```
🟡 审核结果: 有条件通过
有警告但无严重错误，可以选择性修复后发布
```

### 审核未通过 🔴

```
🔴 审核结果: 未通过
有严重错误，必须修复后重新审核
常见错误：
- 数据错误（如NVIDIA营收错误）
- 缺少必要信息
- 敏感内容
```

## 自动化审核（Cron）

### 设置自动审核

```bash
# 每小时自动审核未发布文章
openclaw cron add \
  --name "article-audit" \
  --schedule "0 * * * *" \
  --command "python3 /root/.openclaw/workspace/audit_bot.py"
```

## 责任制度

### 审核责任人

1. **第一责任人**：作者（确保数据准确）
2. **第二责任人**：审核机器人（自动检查）
3. **第三责任人**：发布者（最终确认）

### 错误处理

**发现错误后：**
1. 立即下架错误文章
2. 发布更正声明
3. 记录错误原因
4. 完善审核规则

## 附：审核机器人使用说明

### 安装

```bash

```

### 使用

```bash
# 基本用法
python3 audit_bot.py 文章.md

# 查看帮助
python3 audit_bot.py --help
```

### 输出

- 控制台：实时显示审核结果
- 文件：生成 `.audit.md` 报告文件
- 返回码：0=通过, 1=未通过
