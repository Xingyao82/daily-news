# 无头浏览器 vs API 对比

## 核心区别

| 维度 | API (Brave Search等) | 无头浏览器 (Playwright) |
|------|---------------------|------------------------|
| **工作原理** | 服务商已抓取好的索引 | 自己打开真实浏览器访问 |
| **数据新鲜度** | 有延迟（小时-天） | 实时（当前页面） |
| **限制** | 频率限制、内容过滤 | 受网站反爬限制 |
| **成本** | 免费额度有限 | 完全免费（自己服务器） |
| **可控性** | 只能用API返回的字段 | 可以抓取任何可见内容 |

## 具体对比

### 1. 搜索航班
| | API | 无头浏览器 |
|--|-----|-----------|
| Google Flights | ❌ 没有API | ✅ 直接打开网页抓 |
| 携程/去哪儿 | ❌ 需要商业授权 | ✅ 可以抓取（但有反爬） |
| Skyscanner | ✅ 有API | ✅ 也可以抓网页 |

### 2. 抓取新闻
| | API | 无头浏览器 |
|--|-----|-----------|
| Hacker News | ✅ 有API | ✅ 也可以抓 |
| 任意博客 | ❌ 没有API | ✅ 直接抓 |
| 微信公众号 | ❌ 封闭 | ⚠️ 需要登录 |

### 3. 实时数据
| 场景 | API | 无头浏览器 |
|------|-----|-----------|
| 股票实时价格 | ⚠️ 付费API | ✅ 直接看交易网站 |
| 社交媒体热榜 | ⚠️ 有限制 | ✅ 直接打开看 |
| 电商价格监控 | ❌ 大厂不给API | ✅ 可以抓（需防反爬） |

## 优劣势总结

### API 优势
- ✅ 稳定可靠
- ✅ 结构化数据（JSON）
- ✅ 不担心网站改版
- ✅ 速度快（预处理好）

### API 劣势
- ❌ 有调用限制（1次/分钟）
- ❌ 数据可能过时
- ❌ 很多网站没有API
- ❌ 需要申请Key

### 无头浏览器优势
- ✅ 任何可见的网页都能抓
- ✅ 数据实时
- ✅ 无调用限制
- ✅ 完全免费

### 无头浏览器劣势
- ❌ 可能触发反爬（验证码、封IP）
- ❌ 速度慢（要等页面加载）
- ❌ 网站改版要重写抓取逻辑
- ❌ 需要更多服务器资源

## 最佳实践

**API 适合：**
- 大规模搜索索引
- 需要稳定结构化的数据
- 有官方API的服务

**无头浏览器适合：**
- 没有API的网站
- 需要实时数据
- 抓取动态渲染的内容（JavaScript）
- 模拟用户操作（点击、填写表单）

## 混合策略

实际项目中两者结合用：
1. **先用API** - 快速获取基础数据
2. **再用无头浏览器** - 补充API拿不到的信息
3. **监控用无头浏览器** - 实时检测变化
